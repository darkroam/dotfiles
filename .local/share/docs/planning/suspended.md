# 挂起项

这些项目有意不处于活动状态。仅在满足列出的恢复条件时重新开始；它们不是正常配置的缺陷。

## 状态与硬件

- [ ] `2026-08` 进入本库下一轮检查时，先提醒并询问是否继续跨设备显示管理后续工作；范围包括
  设备适配器、manual marker、统一布局健康、多屏/桌面设备矩阵和长期验收。未再次确认前不自动
  启动修改，本机当前已验证的屏幕切换继续视为完成状态。
- [ ] 建立 `~/.local/share/xdisplay-transition-20260714/` 后，至少稳定使用 14 天并完成 3 次
  冷启动以及开盖、合盖、热插、热拔、延迟外屏和手动布局验证；全部通过且校验恢复清单后，
  还要实际恢复并再次隔离一个无运行影响的文件，确认路径、所有者、mode 和校验和均可还原，
  再删除整个显示迁移隔离目录。执行条件见[显示管理重构计划](display-management-redesign.md)。
- [ ] 仅在 Debian 获得与现有 `simple-mtpfs` 调用接口兼容的 MTP 后端，或完成替代脚本的
  接口和真实 Android 设备验证后，恢复 MTP 挂载审计；普通 USB 与 CIFS 挂载不受影响。
- [ ] 仅在启用更多网络模块或当前两秒天气查询确实造成卡顿时，考虑异步 DWMBlocks 网络模块；
  不复制上游无限重试。
- [ ] 选定可用后端后统一亮度控制。本机没有标准 `/sys/class/backlight` 设备；届时协调
  `xlight`、`xbacklight` 和状态栏动作。
- [ ] 仅在暂停控制必须覆盖非 shell 启动的 MPV 时，加入全局 MPV IPC；使用小型跟踪包装器，
  不采用未经审查的上游 Lua 子模块。
- [ ] 仅在选择符号、维护数据源和可接受网络更新策略后，考虑市场行情模块。
